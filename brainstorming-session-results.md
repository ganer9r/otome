## 핵심 게임 시스템

### 1. 조합 시스템

**기본 구조:**
```
재료 A + 재료 B + 조리도구 = 요리 결과
```

**결과 타입:**
- **성공 (70%)**: 정상적인 요리 (판매가 높음)
- **실패 (25%)**: 이상한 요리 (판매가 낮음, 유머)
- **완전실패 (5%)**: 혐오 요리 (판매 불가, 강한 유머)

**특징:**
- 같은 조합에서 다른 요리 나올 수 있음 (확률)
- 다층 조합 가능 (요리 → 재료로 재사용)
- 30개 레시피 (MVP 기준)

---

### 2. 로그라이트 시스템

**런 구조:**
- 초기 자본: 2,000원
- 재료 구매 → 요리 → 판매 → 반복
- 자본 0원 도달 시 파산 (런 종료)

**세금 시스템:**
- 판매가의 20% 세금
- 매 10턴마다 징수
- **세금 액수 숨김** (심리적 긴장감)
- 턴 카운터만 표시

**메타 진행:**
- 발견한 레시피 영구 저장 (도감)
- 스타 획득 → 영구 업그레이드
- 3가지 업그레이드:
  - 초기 자본 증가
  - 재료비 할인
  - 판매가 상승

---

### 3. 게임 루프

**기본 사이클 (~30초):**
```
1. 재료 선택 (10초)
   ↓
2. 조리도구 선택 (2초)
   ↓
3. 요리 (10초 고정)
   ↓
4. 결과 확인
   ↓
5. [광고 삽입] (랜덤 2-5분 간격)
   ↓
6. 다시하기 / 판매
```

**세션 시간:**
- 회당 ~30초
- 10회 반복 = 5분
- 광고 eCPM 최적화 구간

## 수익화 전략

### 광고 시스템

**타이밍:**
- 요리 완료 → 결과 확인 → **[광고]** → 계속하기
- 랜덤 간격 (2-5분) - 패턴 예측 방지

**오클릭 최적화:**
```
[확인] 버튼 클릭
    ↓
광고 전면 표시
    ↓
광고 종료 후
    ↓
[계속하기] 버튼 (동일 위치)
```
- 버튼 위치 동일 → 연속 클릭 유도
- 광고 터치 확률 증가
- 목표 CTR: 10-12%
- 목표 eCPM: $10-20

**세션 최적화:**
- 짧은 사이클로 자주 플레이 유도
- 중독성 있는 조합 발견 메커니즘
- 메타 진행으로 재방문 유도

## 기술 스택 결정

### 최종 선택: **SvelteKit + React Native (WebView)**

**구조:**
```
React Native (네이티브 앱)
├─ 광고 SDK (LevelPlay/AdMob)
├─ 푸시 (OneSignal)
├─ 분석 (Firebase, Airbridge)
└─ WebView
   └─ SvelteKit 앱 (UI + 게임 로직)
      └─ 라이브 업데이트 가능
```

---

### 선택 이유

**1. 라이브 업데이트**
- 웹앱 부분만 업데이트
- 앱스토어 심사 불필요 (핫픽스)
- 빠른 버그 수정 및 밸런싱

**2. 개발 속도**
- SvelteKit으로 빠른 UI 개발
- 현재 프로젝트 코드 재사용
- 웹 DevTools 사용 가능

**3. 광고 SDK 지원**
- RN: LevelPlay, AdMob 공식 지원
- Capacitor: LevelPlay 지원 없음

**4. 안정성**
- React Native: Meta 지원, 대규모 생태계
- 서드파티 통합 풍부

---

### 기술 스택 상세

**프론트엔드 (웹):**
- SvelteKit 5
- TypeScript
- Svelte Stores (상태관리)

**모바일 (네이티브):**
- React Native
- react-native-webview
- TypeScript

**광고:**
- LevelPlay / AdMob Mediation

**분석/마케팅:**
- Firebase Analytics
- Airbridge (어트리뷰션)
- OneSignal (푸시)

**배포:**
- SvelteKit: Static build → CDN/서버
- React Native: iOS App Store, Google Play

---

### 네이티브 ↔ 웹 통신

**웹 → 네이티브:**
```javascript

window.ReactNativeWebView.postMessage(JSON.stringify({
  type: 'SHOW_AD',
  payload: {}

```

**네이티브 → 웹:**

### Insights & Learnings

**게임 디자인:**
- **예측 불가능성 = 재미**: 같은 조합도 다른 결과, 계속 시도하게 만듦
- **숨겨진 정보 = 긴장감**: 세금 액수 숨김으로 서프라이즈 요소
- **짧은 사이클 = 중독성**: 30초 사이클로 "한 번만 더" 유도

**수익화:**
- **심리적 UI = eCPM**: 버튼 위치 설계로 오클릭 10%+ 가능
- **랜덤 타이밍 = 장기 플레이**: 패턴 예측 방지로 이탈 감소
- **세션 시간 최적화**: 5분 세션으로 광고 노출 극대화

**기술 선택:**
- **하이브리드 전략**: 네이티브 안정성 + 웹 개발 속도 둘 다 확보
- **라이브 업데이트 가치**: 밸런싱 패치 즉시 반영 = 운영 효율
- **생태계 중요성**: 광고 SDK 지원 여부가 기술 선택 결정적 요인

**MVP 스코프:**
- **콘텐츠 적정선**: 30 레시피로 충분한 발견 재미
- **단계적 확장**: v1.0 → v1.1 명확한 로드맵
- **핵심 집중**: 조합 발견 재미가 게임의 본질
