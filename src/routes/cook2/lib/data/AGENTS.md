# 요리 조합 시스템 Guidelines

## 등급 설계

- 요리는 재료와 요리로 구분이 됨.
- 같은 등급의 재료를 합쳐서 한단계 상위 등급 요리를 만듬.
- C급 이상에서는 재료라고 해도 실제 요리여야함.
- G -> R 급까지 설계

### 원칙

- 같은 등급의 재료를 합쳐서 한단계 상위 등급 요리를 만듬.
- 같은 재료로도 요리를 만들 수 있음.
- 실제 존재하는구체적인 코스는 가능(한정식, 오마카세, 가이세키)
- 요리에 부위의 명칭이 들어가는 것은 허용
  - (안심❌, T본스테이크⭕️)

### ❌ 하지 말 것

- 수식어 금지 (상급, 로얄, 황제, 골든..)
- 실제 존재하지 않는 코스 금지 (정식 -> 한정식 -> 코스요리)
- 세부부위 재료 금지 (안심, 등심, 와규)
- 중복 조합 금지 (A+B와 B+A는 같은 조합)
- 하위 등급에 이미 존재하는 요리명 금지

### 파일 참조

recipes.csv 는 추천 할 요리에 참조할 정보로 사용

- 재료 : ingredients-data.ts
- 레시피 : recipes-data.ts
- 참조 : recipes.csv

### 재료 사용 비율 밸런스

- 같은 등급 재료들의 상위 등급 조합 사용 횟수는 균등하게 유지
- 목표 비율: 각 재료당 ±5% 이내
- 특정 재료가 과도하게 많이/적게 사용되지 않도록 검증

## 작성 후 검증(회고)

- 재료+재료=요리일때 요리가 재료과 연관이 있는가?
- 같은 요리가 하위 등급에 존재 하는가?
- 새로 생성한 재료가, 상위등급 재료의 필수 재료에 속하는가?
