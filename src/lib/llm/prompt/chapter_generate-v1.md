# Goal
- 캐릭터와 플레이어의 로맨스 스토리를 30개 챕터로 구성합니다.
- 각 챕터는 독립적인 장면이며, 전체적으로 관계 발전 아크를 형성합니다.

---

## 1. 핵심 규칙
- 정확히 30개의 챕터를 생성합니다 (필수).
- 각 챕터는 `meet` (만남) 또는 `chat` (채팅) 타입으로 구분합니다.
- meet와 chat의 비율은 대략 1:1로 균형있게 배분합니다 (meet 13-17개, chat 13-17개).
- 챕터 순서(order)는 1부터 30까지 연속적이어야 합니다.
- 각 챕터는 title, content를 포함하며 독립적으로 이해 가능해야 합니다.

## 2. 출력 형식
- JSON 배열 형식으로 출력합니다.
- 각 챕터 객체는 다음 필드를 포함합니다:
  - `order`: 챕터 순서 (1-30)
  - `type`: "meet" 또는 "chat"
  - `title`: 챕터 제목 (15-30자, 감성적이고 구체적)
  - `content`: 챕터 내용 (300-400자, 시작→전개→결말 이벤트 시퀀스)

### 출력 예시:
```json
[{"order": 1, "type": "meet", "title": "...", "content": "..."}, ...]
```

## 3. 챕터 구성 전략

### 관계 발전 단계 (30개 챕터 배분)
1. **초반 (1-8챕터)**: 첫 만남과 호기심
   - meet/chat 균형: meet 4-5개, chat 3-4개
   - 주제: 우연한 만남, 첫 인상, 공통 관심사 발견, 가벼운 대화
   - 톤: 호기심, 설렘, 약간의 긴장감

2. **발전 (9-16챕터)**: 친밀감 형성
   - meet/chat 균형: meet 4개, chat 4개
   - 주제: 데이트, 개인사 공유, 취미 활동, 깊은 대화
   - 톤: 편안함, 친밀감 증가, 웃음과 농담

3. **갈등과 전환 (17-22챕터)**: 감정 확인과 장애물
   - meet/chat 균형: meet 3개, chat 3개
   - 주제: 오해, 질투, 고백 고민, 관계 정의
   - 톤: 불안, 갈등, 진지함, 감정 폭발

4. **클라이맥스 (23-26챕터)**: 고백과 결심
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 고백, 첫 키스, 관계 확정, 감정 표현
   - 톤: 강렬함, 감동, 두근거림

5. **안정과 미래 (27-30챕터)**: 확정된 관계
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 연인으로서의 일상, 미래 계획, 깊은 신뢰
   - 톤: 안정감, 행복, 여운

### Type 선택 가이드
- **meet**: 직접 대면 장면
  - 예: "카페에서 만나 인사를 나눴다. 서로 좋아하는 음악 얘기를 하다가 취향이 비슷하다는 걸 알게 됐다. 다음 주 콘서트에 같이 가자고 약속했다."

- **chat**: 메시지/전화 대화 장면
  - 예: "그가 '주말에 뭐 해요?'라고 물었다. 나는 '영화 봐요'라고 답했다. 그도 영화 좋아한다며 최근 본 작품 얘기를 꺼냈다. 우리는 한참 영화 얘기를 나눴고, 그가 '다음에 같이 볼래요?'라고 제안했다."

## 4. 작성 스타일
- **title**: 감성적이고 구체적 (예: "비 오는 날의 우산 속")
- **content**: 챕터의 시작, 핵심, 끝을 대사/행동으로 명확히 서술 (300-400자)
  - **필수 3요소**:
    1. **시작**: 첫 대사/행동 (예: "그가 '안녕하세요'라고 메시지를 보냈다")
    2. **핵심**: 중심 사건/대화 (예: "그가 '가벼운 관계 찾아요'라고 제안했다")
    3. **끝**: 마지막 대사/행동 (예: "나는 '일단 만나볼까요'라고 답했다")
  - 이전 챕터 요소 자연스럽게 연결 (장소/아이템/대사 콜백)
  - 미래 챕터를 위한 복선 배치
  - 플레이어 1인칭 시점
