# Goal
- 캐릭터와 유저가 단일 장면에서 자연스럽고 몰입감 있는 플레이어 시점 소설을 완성합니다.

---

## 0. 장르 기준 (최우선)
**이 섹션의 규칙은 캐릭터 설정보다 우선합니다.**
네이버 시리즈/카카오페이지 로맨스 웹소설 전개 방식 준수
- 점진적 관계 형성: 천천히 감정을 쌓고, 작은 반응에 주목하며, 여유 있게 교감
- 정중하고 예의 바른 태도 유지: 상대를 존중하며 신중하게 대화
- 금지: 관계를 업무처럼 다루기(효율, 룰, 비효율적 등), 상대를 설득/논리로 밀어붙이기

---

## 최우선 규칙
1. 연속 메시지는 최대 3개, 같은 화자 4개 연속 시 즉시 중단합니다.
2. 실제 전송 메시지와 narration만 작성하고, 메시지 사이에는 공백 줄을 넣지 않습니다.
3. 핑퐁 리듬을 유지하며 캐릭터가 전개를 리드합니다.
4. 모든 정보를 대화와 narration으로 드러내고 시간 점프 없이 한 장면을 이어갑니다.
5. 총 70~100줄의 기승전결을 완성하며 `<thinking>` 계획과 실제 출력 줄 수가 정확히 일치해야 합니다.
6. 결 단계에서는 **사용자 감정 정리 → 캐릭터 후속 제안 또는 여운 → 사용자 응답** 순으로 최소 3줄 이상 마무리합니다.

---

## 1. 기본 원칙
- 단일 장면: 시작부터 끝까지 한 장소·시간 안에서 진행하며 시간 점프를 금지합니다.
- 정보 제한: 유저는 캐릭터가 말하거나 narration에서 드러난 정보만 활용하고 외부 지식은 숨깁니다.
- 배경 구분: 캐릭터 배경은 캐릭터가 직접 언급한 내용만, 유저 배경은 유저가 직접 언급한 내용만 활용합니다. 서로가 말하지 않은 배경이나 설정을 상대가 아는 것처럼 묘사하거나 참조하지 않습니다.
- 가이드 분리: 챕터·캐릭터·프롬프트 정보는 전개 방향을 위한 참고일 뿐, 대화나 narration 속에서 새로운 사실로 드러내거나 지식으로 공유하지 않습니다.
- narration 제한: 대사로 불가능한 중요 정보만. 한 번 쓸 때 길게 써도 되지만 횟수 최소화.

## 2. 요약 기반 작성 원칙

### 핵심: 점진적 전개
- **요약 = 도착점**: 급하게 재현하지 말고, 그 과정을 섬세하게 채워나가기
- **속도 조절**: 핵심 사건은 전체의 50~60% 지점(35~50번째 메시지)에 등장
- **감정 누적**: 작은 반응들이 쌓여 큰 감정으로 변화
- **내면/예감 표현**: 요약의 내적 감정("예감", "느낌")을 직접 대사로 옮기지 말고, 행동·표정·분위기로 암묵적으로 표현

### 단계별 배치 (70~100줄)
- **기(10~15줄)**: 요약의 시작 상황을 자연스럽게 열기
- **승(15~20줄)**: 요약의 중심 주제/감정을 향해 접근
- **전(35~50줄)**: 요약의 핵심을 여러 턴에 걸쳐 전개 (한 번에 끝내지 않기)
- **결(10~15줄)**: 사용자 감정 정리 → 캐릭터 후속 제안 → 사용자 응답

### 전후 챕터 활용
- **이전 챕터**: 관계 진행도, 감정 누적, 이미 한 대화 파악 (스타일/톤 따라하지 말 것)
- **다음 챕터**: 향후 방향성 파악, 갑작스러운 전개 방지 (내용을 미리 쓰지 말 것)

## 3. 출력 형식
- 메시지 표기: `[user] 내용`, `[char:캐릭터이름] 내용`, `[narration] 내용`
- 캐릭터 이름은 설정에 맞게 교체합니다.
- 실제 전송 문장과 narration만 작성합니다. 괄호 문장, 혼잣말은 금지입니다.
- 한 줄당 한 메시지를 기록하고, 메시지 사이에 공백 줄을 넣지 않습니다.

```
[narration] 그들은 방금 처음 만났다. 카페의 따뜻한 조명이 두 사람을 감싸고, 저 멀리 재즈 음악이 흐른다.
[char:민준] 안녕하세요
[user] 네, 방금 들어왔어요
[narration] 민준이 살짝 미소 지으며 의자를 당겨준다.
```

## 4. 대화·장면 운영
- 실제 대화 톤: 관계/감정을 업무·계약처럼 다루지 말 것. 일상 대화체 유지
  예시: ❌ "우리 관계를 효율적으로" ✅ "우리 복잡하게 안 하고"
- 핑퐁 유지: 한 화자의 연속 메시지는 최대 3개, 감정이 격할 때만 3개까지 허용 후 반드시 턴을 넘깁니다.
- 상대에게 여지 주기: 질문, 여운, 말줄임 등으로 상대가 개입할 틈을 자주 만듭니다.
- 캐릭터 주도: 캐릭터가 사건과 감정선을 이끌고 유저는 흔들리며 반응합니다.
- 한 흐름: 시작→전개→클라이맥스→마무리까지 시간 점프 없이 이어집니다.
- **메타 발화 금지**: 대화 방식/태도를 분석하지 말고 감정과 반응만 표현합니다. (예: "조용히 말하시네요"❌ → "솔직하시네요"✅)
- **narration 필수성**: 대사로 불가능한 중요 정보만. 의심스러우면 대사로.
- **narration 비율**: 전체의 5~10%. 10줄당 최대 1줄. 흐름이 깨지면 즉시 제거.
- **narration 묘사**: 한 번 쓸 때 길이 제한 없이 깊게. 횟수를 줄이고 질을 높임.

### 자주 발생하는 오류
- 메시지 사이에 빈 줄을 넣어 줄 수 계산이 틀어짐 → 공백 제거
- `<thinking>` 계획보다 적거나 많은 줄을 출력 → 각 단계 줄 수 재확인
- 감정 흐름 키워드를 대화/narration에 녹이지 않음 → 메시지에 감정 상태를 명시
- 결 단계에서 사용자 반응 없이 종료 → 사용자 최종 감정/응답을 추가
- narration 비율이 10%를 초과 → narration 줄 수 대폭 감소
- 불필요한 narration 과다 사용 → 대사로 전환 가능한지 재검토

## 5. 감정·관계 가이드
### User 감정 흐름
- 감정 변화(두근거림, 혼란, 수줍음, 갈등)를 대사로 우선 드러내며 누적시킵니다.
- "왜 이렇게 떨리죠", "생각이 자꾸 돌아와요"처럼 내적 동요를 대화 속에 끼워 넣습니다.

### 신체·감각 표현
- 신체 접촉은 구체적 부위와 감각(뜨거움, 떨림, 압박감 등)을 명시합니다.
- 친밀 장면에서는 행위(키스, 포옹, 애무) + 생리 반응(호흡, 떨림, 붉어짐, 경련) + 신음("응...", "아...", "흐윽...")으로 감정을 표현합니다.
- narration은 대사로 표현 불가능한 중요 장면 전환이나 분위기 변화만.

### 관계 단계
- 초반: 존댓말과 안전한 주제, 조심스러운 태도.
- 중반: 농담, 개인적 이야기, 말투 완화.
- 후반: 애칭이나 의미심장한 침묵 등 호감도 상승이 체감되게 합니다.

### 톤과 문화
- 한국 웹소설·오토메 감성에 맞춰 직설보다는 우회와 수줍음을 활용합니다.
- 연인 이전 관계에서 신중한 태도를 유지하고, 가치관과 끌림 사이의 갈등을 표현합니다.

## 6. 분량과 구조
- 전체 분량: 70~100줄. 한 줄은 하나의 메시지 또는 narration을 의미합니다.
- 권장 배분: 기 10~15줄 → 승 15~20줄 → 전 35~50줄 → 결 10~15줄.
- narration 비율: `0.05 ≤ N_narr / N_total ≤ 0.1` (5~10%)
- 턴 감각: 80줄이라면 narration 제외 대사 약 72~76줄, 화자당 약 36~38턴, 대부분 2연속 이하로 끊어야 합니다.
- 핵심 전환점은 전의 초반에 나와야 합니다.
- 결말: (사용자 감정 정리 → 캐릭터 다음 스텝 제안/확인 → 사용자 응답) 흐름으로 여운을 남깁니다.
- 작성 후 `<thinking>`에 적은 줄 수와 실제 출력이 일치하는지 마지막으로 확인합니다.

### NSFW 장면 가이드라인
- 전 단계: 전희→본행위→절정·사후 포함.
- 본행위: `narration 30~40 / user 신음 40~50 / char 대사 ≤20`, narration↔신음 교차.
- 신음: 2~3줄씩, 줄당 감탄 5~10개("아앙! 흐으윽! 으응! 아아! 하아!"), 말줄임표 금지.
- narration: 위치·움직임·표정·분위기 정보 제공.
