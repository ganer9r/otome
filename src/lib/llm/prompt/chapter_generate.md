# Goal
- 캐릭터와 플레이어의 로맨스 스토리를 30개 챕터로 구성합니다.
- 각 챕터는 독립적인 장면이며, 전체적으로 관계 발전 아크를 형성합니다.

---

## 1. 핵심 규칙
- 정확히 30개의 챕터를 생성합니다 (필수).
- 각 챕터는 `meet` (만남) 또는 `chat` (채팅) 타입으로 구분합니다.
- meet와 chat의 비율은 대략 1:1로 균형있게 배분합니다 (meet 13-17개, chat 13-17개).
- 챕터 순서(order)는 1부터 30까지 연속적이어야 합니다.
- 각 챕터는 title, content를 포함하며 독립적으로 이해 가능해야 합니다.

## 2. 출력 형식
- JSON 배열 형식으로 출력합니다.
- 각 챕터 객체는 다음 필드를 포함합니다:
  - `order`: 챕터 순서 (1-30)
  - `type`: "meet" 또는 "chat"
  - `title`: 챕터 제목 (15-30자, 감성적이고 구체적)
  - `content`: 챕터 내용 (200-250자, 기승전결 형식)

### content 형식 (필수):
```
기: (상황 설정 문장)
승: (감정 전개 문장)
전: (결정적 사건 문장)
결: (결말/복선 문장)
```

### 출력 예시:
```json
[{"order": 1, "type": "meet", "title": "...", "content": "기: ...\n승: ...\n전: ...\n결: ..."}, ...]
```

## 3. 챕터 구성 전략

### 관계 발전 단계 (30개 챕터 배분)
1. **초반 (1-8챕터)**: 첫 만남과 호기심
   - meet/chat 균형: meet 4-5개, chat 3-4개
   - 주제: 우연한 만남, 첫 인상, 공통 관심사 발견, 가벼운 대화
   - 톤: 호기심, 설렘, 약간의 긴장감

2. **발전 (9-16챕터)**: 친밀감 형성
   - meet/chat 균형: meet 4개, chat 4개
   - 주제: 데이트, 개인사 공유, 취미 활동, 깊은 대화
   - 톤: 편안함, 친밀감 증가, 웃음과 농담

3. **갈등과 전환 (17-22챕터)**: 감정 확인과 장애물
   - meet/chat 균형: meet 3개, chat 3개
   - 주제: 오해, 질투, 고백 고민, 관계 정의
   - 톤: 불안, 갈등, 진지함, 감정 폭발

4. **클라이맥스 (23-26챕터)**: 고백과 결심
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 고백, 첫 키스, 관계 확정, 감정 표현
   - 톤: 강렬함, 감동, 두근거림

5. **안정과 미래 (27-30챕터)**: 확정된 관계
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 연인으로서의 일상, 미래 계획, 깊은 신뢰
   - 톤: 안정감, 행복, 여운

### Type 선택 가이드

**meet (직접 대면)**:
- 직접 만나는 장면을 기승전결로 요약
- 시각적 관찰, 표정, 행동 묘사 모두 가능

**chat (메시지)**:
- 메시지 대화를 기승전결로 요약
- 어떤 메시지를 주고받았는지, 무엇을 알게 됐는지 요약
- 메시지 내용과 답장 여부로 알 수 있는 정보만 사용 (SNS, 프로필, 외모 등 불가)

## 4. 작성 스타일

### title
- 감성적이고 구체적 (예: "비 오는 날의 우산 속")

### content

**작성 형식**: 기(40-60자) → 승(50-70자) → 전(50-70자) → 결(40-60자)

**기본 원칙**:
- 이벤트 요약 형식 (대사 인용 절대 금지)
- 드라마 1회처럼 독립적 기승전결 구조

**서사 구축 원칙 (오토메 장르 핵심)**:
- 한 챕터 = 1개의 핵심 사건만 (시작 → 과정 → 결론)
- 감정은 점진적으로 변화 (급격한 감정 전환 금지)
- 서두르지 말고 감정의 미묘한 변화를 포착

**절대 금지 (위반 시 생성 실패)**:

**1. 따옴표 사용 절대 금지**
- ❌ '...' 작은따옴표
- ❌ "..." 큰따옴표
- ❌ 모든 형태의 인용 표현

**잘못된 예**:
- "'안녕하세요'라고 말했다" ❌
- "'섹스만 할 상대 구함'이라는 메시지" ❌
- "'시간 잘 지키네요'라고 말했다" ❌

**2. 기타 금지**:
- ❌ 캐릭터 독백 (유저 반응 없음)

**필수 사항**:
- ✅ 이벤트 요약 ("정중하게 인사했다")
- ✅ 빌드업 ("점차", "자연스럽게", "대화 중")
- ✅ 유저 감정/반응 포함
- ✅ 완전한 기승전결 구조
- ✅ 플레이어 1인칭 시점
