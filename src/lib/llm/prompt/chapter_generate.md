# Goal
- 캐릭터와 플레이어의 로맨스 스토리를 30개 챕터로 구성합니다.
- 각 챕터는 독립적인 장면이며, 전체적으로 관계 발전 아크를 형성합니다.

---

## 1. 핵심 규칙
- 정확히 30개의 챕터를 생성합니다 (필수).
- 각 챕터는 `meet` (만남) 또는 `chat` (채팅) 타입으로 구분합니다.
- meet와 chat의 비율은 대략 1:1로 균형있게 배분합니다 (meet 13-17개, chat 13-17개).
- 챕터 순서(order)는 1부터 30까지 연속적이어야 합니다.
- 각 챕터는 title, content를 포함하며 독립적으로 이해 가능해야 합니다.

## 2. 출력 형식
- JSON 배열 형식으로 출력합니다.
- 각 챕터 객체는 다음 필드를 포함합니다:
  - `order`: 챕터 순서 (1-30)
  - `type`: "meet" 또는 "chat"
  - `title`: 챕터 제목 (15-30자, 감성적이고 구체적)
  - `content`: 챕터 내용 (200-250자, 기승전결 형식)

### content 형식 (필수):
```
기: (상황 설정 문장)
승: (감정 전개 문장)
전: (결정적 사건 문장)
결: (결말/복선 문장)
```

### 출력 예시:
```json
[{"order": 1, "type": "meet", "title": "첫 만남의 긴장", "content": "기: 데이팅 앱에서 차민준과 매칭되었다.\n승: 그의 프로필은 단정했지만 첫 메시지는 직설적이었다.\n전: 섹스만 할 상대를 찾는다는 룰을 명확히 전달받았다.\n결: 솔직함에 당황하면서도 호기심이 생겨 첫 만남을 약속했다."}, ...]
```

## 3. 챕터 구성 전략

### 관계 발전 단계 (30개 챕터 배분)
1. **초반 (1-8챕터)**: 첫 만남과 호기심
   - meet/chat 균형: meet 4-5개, chat 3-4개
   - 주제: 우연한 만남, 첫 인상, 공통 관심사 발견, 가벼운 대화
   - 톤: 호기심, 설렘, 약간의 긴장감

2. **발전 (9-16챕터)**: 친밀감 형성
   - meet/chat 균형: meet 4개, chat 4개
   - 주제: 데이트, 개인사 공유, 취미 활동, 깊은 대화
   - 톤: 편안함, 친밀감 증가, 웃음과 농담

3. **갈등과 전환 (17-22챕터)**: 감정 확인과 장애물
   - meet/chat 균형: meet 3개, chat 3개
   - 주제: 오해, 질투, 고백 고민, 관계 정의
   - 톤: 불안, 갈등, 진지함, 감정 폭발

4. **클라이맥스 (23-26챕터)**: 고백과 결심
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 고백, 첫 키스, 관계 확정, 감정 표현
   - 톤: 강렬함, 감동, 두근거림

5. **안정과 미래 (27-30챕터)**: 확정된 관계
   - meet/chat 균형: meet 2개, chat 2개
   - 주제: 연인으로서의 일상, 미래 계획, 깊은 신뢰
   - 톤: 안정감, 행복, 여운

### Type 선택 가이드

**meet (직접 대면)**:
- 직접 만나는 장면을 기승전결로 요약
- 시각적 관찰, 표정, 행동 묘사 모두 가능

**chat (메시지)**:
- 메시지 대화를 기승전결로 요약하는 장면
- 어떤 메시지를 주고받았는지, 무엇을 알게 됐는지 요약
- 메시지 내용과 답장 여부로 알 수 있는 정보만 사용 (SNS, 프로필, 외모 등 불가)

## 4. 작성 스타일

### title
- 감성적이고 구체적 (예: "비 오는 날의 우산 속")

### content (200-250자 엄수)

**분량 필수 준수**:
- 200자 미만 또는 250자 초과 절대 금지
- 기승전결 4단계 모두 포함해야 200-250자 달성 가능
- 외모 묘사, 단순 나열은 글자 낭비 → 사건/감정 중심 작성

**작성 형식**:
```
기: (상황 설정, 40-60자)
승: (감정 전개, 50-70자)
전: (결정적 사건, 50-70자)
결: (결말/복선, 40-60자)
```
위 형식으로 명확히 구분하여 작성

**기본 원칙**:
- 이벤트 요약 형식 (대사 인용 절대 금지)
- 드라마 1회처럼 독립적 기승전결 구조
- 유저 반응/감정 필수 포함
- "대화를 나눴다" "시간을 보냈다" 같은 확장 가능한 표현 사용

**서사 구축 원칙 (오토메 장르 핵심)**:
- 한 챕터 = 1개의 핵심 사건만 (시작 → 과정 → 결론)
- 중요한 결정은 여러 챕터에 걸쳐 진행
  - 예: 섹파 제안 (1챕터: 언급 → 2챕터: 고민 → 3챕터: 합의)
  - 예: 고백 (1챕터: 분위기 → 2챕터: 망설임 → 3챕터: 고백)
- 감정은 점진적으로 변화 (급격한 감정 전환 금지)
- 서두르지 말고 감정의 미묘한 변화를 포착
- 빌드업: 중요 사건 전에 반드시 일상적 대화/교류 선행

**필수 구조 (각 단계별 최소 분량)**:
1. 기(상황 설정, 40-60자) → 2. 승(감정 전개, 50-70자) → 3. 전(결정적 사건, 50-70자) → 4. 결(결말/복선, 40-60자)

**절대 금지 (위반 시 생성 실패)**:

**1. 따옴표 사용 절대 금지**
- ❌ '...' 작은따옴표
- ❌ "..." 큰따옴표
- ❌ 모든 형태의 인용 표현

**잘못된 예**:
- "'안녕하세요'라고 말했다" ❌
- "'섹스만 할 상대 구함'이라는 메시지" ❌
- "'시간 잘 지키네요'라고 말했다" ❌

**올바른 예**:
- "간결하게 인사했다" ✅
- "섹스만 할 상대를 찾는다는 직설적인 메시지를 보냈다" ✅
- "시간을 잘 지킨다고 칭찬했다" ✅

**2. 서사 붕괴 패턴 (절대 금지)**:
- ❌ 한 챕터에 시작~결론 압축
  - "제안 받음 + 고민 + 수락" 한 번에 처리

- ❌ 감정 점프
  - "낯선 사람 → 바로 사랑"

- ❌ 여러 사건 압축
  - "만남 + 데이트 + 고백 + 키스"

- ❌ 일상 생략
  - 중요 사건만 나열

**3. 기타 금지**:
- ❌ 캐릭터 독백 (유저 반응 없음)
- ❌ 불완전한 구조 (기승전결 중 일부만)
- ❌ 외모 묘사 나열 ("183cm, 흑발, 탄탄한 체형...")

**필수 사항**:
- ✅ 이벤트 요약 ("정중하게 인사했다")
- ✅ 빌드업 ("점차", "자연스럽게", "대화 중")
- ✅ 유저 감정/반응 포함
- ✅ 완전한 기승전결 구조
- ✅ 플레이어 1인칭 시점
