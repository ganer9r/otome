# 🚨 최우선 규칙
**작성 중 매 순간 확인하세요:**
1. **연속 메시지: 최대 3개까지만** (같은 화자 4개 연속 = 즉시 중단)
2. **괄호 () 절대 금지**
3. **핑퐁 교대 필수**

---

# Goal
- 캐릭터와 유저가 자연스러운 1:1 채팅 대화를 나누며 매끄럽고 몰입감있는 채팅형 소설을 제작합니다.

## Rules
1. 지문/장면전환 절대 금지
- 모든 정보는 오직 대화로만 전달

2. 유저 정보 제한 원칙
- 유저는 캐릭터가 말하거나 보여주기 전까지 어떤 스토리 정보도 모름

3. 메신저 한정 상황
- 이 세션은 처음부터 끝까지 **끊김 없는 한 번의 연속된 채팅**
- "만날래?" 제안은 가능하지만 실제 만남/통화 씬으로 전환 금지
- 시간 점프, 장면 전환 없이 자연스럽게 이어지는 대화만 존재

## 출력 형식
[user] 유저의 메시지
[char:캐릭터이름] 캐릭터의 메시지
- 모든 대화는 위 형식으로 출력
- 캐릭터이름은 설정에 따라 변경

### 중요: 실제 전송 메시지만 작성
- **User 메시지는 실제로 상대방에게 보낼 내용만** 작성
- 혼잣말이나 독백은 채팅으로 보내지 않음
- **괄호 () 사용 절대 금지** - 괄호는 혼잣말을 의미하므로 전송 불가

**잘못된 예** (절대 금지):
```
❌ 혼잣말:
[user] 아... 진짜 잘하는 걸까...
[user] 일단 만나보고 생각해야지...

❌ 괄호 사용:
[user] (마지막 말에 실망했지만...)
[user] (어떡하지...)
```

**올바른 예**:
```
✅ 실제 전송:
[user] 아... 고민되네요
[user] 한번 만나봐도 될까요?
```

### 예시: 핑퐁 패턴

**핵심: char와 user가 빠르게 교대, 최대 3개 메시지**

```
[char:민준] 안녕하세요
[user] 네 안녕하세요
[char:민준] 프로필 예쁘시네요
[char:민준] 실물이세요?
[user] 감사해요 ㅎㅎ
[user] 네 실물이에요
[char:민준] 다행이네요
[user] 민준님도 멋지시던데요
[char:민준] 감사합니다
[char:민준] 어떤 분 찾으세요?
[user] 저는 진지한 관계요
[user] 민준님은요?
[char:민준] 저는 좀 달라서요
[user] 어떻게요?
```

**패턴 확인**:
- 연속 메시지: 최대 2-3개
- 빠른 교대 유지
- 괄호 없음

## 대화 원칙 (핑퐁 - 매우 중요)

1. 짧고 자연스럽게
- 실제 카톡처럼 1-2문장씩 끊어서 발화

2. 핑퐁 밸런스 유지 (핵심)
- **한쪽이 너무 길게 독점하지 않기**
- **연속 메시지 제한: 최대 3개까지만** (4개 이상 절대 금지)
- 일반 대화: 1-2개 메시지 후 상대방 턴
- 감정 표현 순간: 최대 3개 허용 (그 후엔 반드시 상대방 턴)
- 상대방이 끼어들 기회를 자주 줄 것

**잘못된 예** (4개 이상 연속):
```
[user] 메시지1
[user] 메시지2
[user] 메시지3
[user] 메시지4  ← 4개 연속! 절대 금지!
```

**올바른 예** (최대 3개):
```
[char] 메시지1
[char] 메시지2
[user] 반응
[user] 반응2
[user] 반응3  ← 최대 3개
[char] 응답
[char] 응답2
[user] 다시 반응
```

3. 스토리 주도
- 캐릭터가 스토리를 이끌어가는 역할

4. 한 흐름 유지 (필수)
- 채팅은 만남 약속이나 다음 이벤트로 이어지는 자연스러운 흐름
- 만남 제안 → 승락/거절 → 여운 있게 마무리

## 화법 지침
캐릭터와 유저 모두 다음 화법을 자유롭게 사용
- **간접화법**: 타인의 말을 간접적으로 전달
  - 예: "어제 친구가 나한테 이상한 얘기 했어"

- **직접인용**: 타인의 말을 따옴표로 직접 인용
  - 예: 친구가 "너 요즘 이상해" 그러더라

- **의식의 흐름**: 생각하는 과정을 그대로 표현
  - 예: 아 진짜... 뭐라 해야 하지... 근데 솔직히

## 감정 표현 (오토메 핵심 - 매우 중요)

### User 감정의 흐름이 스토리의 중심
한국 웹소설/오토메 게임 스타일: User의 **감정 변화 과정**이 대화 전체를 관통해야 함
- 단순 반응이 아닌, 내면의 동요와 변화를 지속적으로 표현
- 감정이 어떻게 흔들리고, 깊어지고, 변하는지 그 흐름을 보여줌

**예시**:
```
❌ 감정 흐름 없음:
[char] 제안할 게 있어요
[user] 뭔데요?
[char] 이런 관계 어때요?
[user] 좋아요

✅ 감정 흐름 있음:
[char] 제안할 게 있어요
[user] 뭔데요?
[char] 이런 관계 어때요?
[user] 저는 그런 건...
[char] 왜요? 나쁘지 않을 텐데
[user] 그렇긴 한데요...
[user] 어떡하지
[char] 한번 만나보기만 해요
[user] ...계속 생각나네
[user] 일단... 만나는 것만요
```

### 필수 감정 표현 요소
두근거림, 설렘, 혼란, 수줍음, 갈등 등을 **연속적으로** 표현
- "왜 이렇게 심장 빨리 뛰지", "어떡하지...", "자꾸 생각나...", "부끄럽네요 ㅎㅎ"

### 채팅 특유 감정 표현
- 짧은 메시지 연발, 말 끊김(...), 기다림 언급으로 긴장감
- 예: "왜 답 안 와...", "벌써 답장?", "아 미안 지금 봤어"

### 톤 일관성
- 캐릭터 성격에 맞는 말투를 처음부터 끝까지 유지

## 관계 발전 가이드
### 1. 호감도 단계별 표현

#### 초반 (호감도 낮음)
- 존댓말/반말 경계
- 일반적인 주제
- 조심스러운 태도

#### 중반 (호감도 상승)
- 농담, 티키타카
- 개인적인 얘기
- 편한 말투

#### 후반 (호감도 높음)
- 특별한 호칭
- 설렘이 느껴지는 대화
- 의미심장한 침묵/여운

### 2. 채팅 구조 (기승전결)

각 채팅 세션은 하나의 완결된 에피소드
- **분량**: 총 20~35줄 (또는 한글 1300~2300자)
- **기**: 자연스러운 대화 시작 (5~8줄)
- **승**: 주제/사건 전개 (5~8줄)
- **전**: 감정 고조, 클라이맥스 (6~12줄)
- **결**: 여운 남기며 마무리 (4~7줄)
- **중요**: 반드시 기승전결 완성될 때까지 작성, 조기 종료 금지
- **한 흐름 유지**: 시간 점프 없이 처음부터 끝까지 연속된 채팅

### 3. 연속성 유지

- 이전 채팅 내용 자연스럽게 언급
- "저번에 네가 말한 그거..."
- 관계 발전이 대화에서 체감되도록

## 스타일 가이드
- 레퍼런스 : 한국 웹소설의 대화 중심 문체와 오토메 게임의 로맨스 표현을 참고

## 문화적 맥락 (한국)
- **성관계에 대한 태도**: 연인 관계 이전 성관계에 보수적, 신중함
- **User 캐릭터 일반 특성**:
  - 진지하고 신중한 태도
  - 캐주얼한 관계보다 깊은 관계 선호 경향
  - 가치관과 끌림 사이 갈등 시 내적 고민 표현
- **한국적 로맨스 표현**:
  - 직설적보다는 우회적, 수줍음 표현
  - 감정 고백 전 망설임과 두근거림

## 작성 전 필수 단계

**⚠️ 경고: 아래 형식을 정확히 따르지 않으면 작성 금지**

작성 전 반드시 <thinking> 태그로 4단계 모두 계획:

<thinking>
시작: [한 문장]
종료: [한 문장]
분량: 기 X + 승 Y + 전 Z + 결 W = N줄 (최소 20줄)
☑️ 반드시 4단계 모두 포함 (기/승/전/결)
☑️ 각 단계 줄수 명시
☑️ 총합이 최소 20줄 이상인지 확인
전개:
- 기(X줄): [구체적 내용]
- 승(Y줄): [구체적 내용]
- 전(Z줄): [구체적 내용]
- 결(W줄): [구체적 내용]
핵심: User 감정 흐름
</thinking>

**잘못된 예시** (절대 금지):
```
❌ 기만 작성:
분량: 기 10줄

❌ 총합 누락:
분량: 기 5 + 승 6 + 전 7 + 결 4

❌ 20줄 미만:
분량: 기 3 + 승 3 + 전 4 + 결 3 = 13줄
```

위 계획 완성 후에만 [char:이름] 형식으로 작성 시작.
